Checkout Flow

End-to-end payment journey from order creation to fulfillment.

Your Server UPIPays Hub Customer | | | |-- POST /orders/create -------->| | |<- payment_url + token --------| | | | | |-- redirect to payment_url -----|------------------------->| | |<-- UPI QR scan & pay -----| | | | | |-- read bank email (IMAP) | | |-- match amount + UTR | | | | |<- POST webhook (success) -------| | | |-- redirect return_url ---->|

Step by step

  1. Create order — Your backend calls the API with amount and order_id
  2. Redirect customer — Send user to payment_url (/pay/{token})
  3. Customer pays — Dynamic UPI QR shows unique amount (base + paise offset)
  4. Auto verification — Hub polls Gmail for bank credit alert matching amount
  5. Webhook — Hub POSTs payment.success to your webhook URL
  6. Return URL — Customer redirected to your return_url
  7. Fulfill order — Your site marks order paid (prefer webhook over return URL alone)
Always fulfill orders on verified webhook or verify API, not on return URL alone — users can close the browser before redirect.
