Authentication

All API requests use HMAC-SHA256 request signing with your merchant API key and secret.

Required headers

HeaderDescription
X-Merchant-KeyYour API key (starts with mk_)
X-TimestampUnix timestamp in seconds (UTC). Must be within ±15 minutes of server time.
X-SignatureHex-encoded HMAC-SHA256 signature
Content-Typeapplication/json for POST requests

Signature algorithm

Build the message string:

message = timestamp + "|" + method + "|" + path + "|" + body
  • timestamp — same value as X-Timestamp header
  • method — uppercase HTTP method (GET, POST)
  • path — request path only, e.g. /api/v1/orders/create (no domain, no query string)
  • body — raw JSON body string, or empty string for GET
signature = HMAC_SHA256(api_secret, message) → hex encode

Example (create order)

timestamp = "1717654321"
method    = "POST"
path      = "/api/v1/orders/create"
body      = '{"order_id":"ORD-1","amount":100,...}'

message   = "1717654321|POST|/api/v1/orders/create|{...}"
signature = hmac_sha256(api_secret, message)
Important: Sign the exact raw body bytes you send. Whitespace and field order matter.
