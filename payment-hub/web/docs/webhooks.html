Webhooks

UPIPays sends a POST request to your webhook URL when payment status changes.

Events

EventWhen
payment.successPayment verified successfully
payment.failedPayment failed

Payload

{
  "event": "payment.success",
  "hub_order_id": "HUB-000042",
  "order_id": "INV-1001",
  "amount": 499.00,
  "currency": "INR",
  "status": "success",
  "paid_at": "2026-06-06T12:15:00Z"
}

Webhook headers

HeaderDescription
X-Hub-EventEvent name (e.g. payment.success)
X-Hub-TimestampUnix timestamp when webhook was sent
X-Hub-SignatureHMAC-SHA256 hex signature

Verify webhook signature

Same algorithm as API auth, but path is the URL path of your webhook endpoint:

message = timestamp + "|POST|" + webhook_path + "|" + raw_body
expected = HMAC_SHA256(api_secret, message)

Example: if webhook URL is https://yoursite.com/payment/webhook, path is /payment/webhook.

Handler checklist

  1. Read raw request body before parsing JSON
  2. Verify X-Hub-Signature using your API secret
  3. Check order_id matches an order you created
  4. Verify amount matches expected value
  5. Return HTTP 200 quickly — do heavy work async
  6. Make handler idempotent (webhooks may retry)
